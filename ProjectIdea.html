Project Idea

Overview

FlickDeck is an app developed to streamline decisions of what content on various streaming platforms to consume by linking user accounts in order to create a shared list of “liked” content chosen in a swipe left or right method made popular by applications such as Tinder.

Motivation

Content streaming services have become the new norm for most households preferred method of entertainment, in Australia alone Netflix has 11.9 million subscribers and when it comes to deciding what to watch and 2016 American study found that on average users spend 17.8 minute browsing Netflix before choosing something to watch, and that 40 percent of users wanted to watch something deferent to their partner or spouse. While curator and aggregator apps such as reelgood and dinggo, (of which are unavailable to the Australian market at this stage) exist they do not focus on issue of partners not agreeing on something to watch where as FlickDeck’s specific purpose is to curate a list for a selected linked user group for consumption.

Description

FlickDeck could crudely be summed up as “Tinder for movies” in which users will be shown the poster/title card for a movie or show and the user is prompted to either swipe right if they are interested in the content or left if they are not interested. The main function of the app is that users will link their account with another user account, for example a partner or spouse, both of the users with linked accounts will be shown the same content on their separate devices. when users have both swiped right the selected piece of content will be added to a list that the users have access to and they will be alerted to the match. The content the app displays will be taken from a database of content that is available on selected streaming services (Netflix, Amazon, Stan etc.) that the users will indicate they have access to when creating an account.

The overall functionality and user experience within the app itself is purposely simple in an effort to have a streamlined decision process when users sit down to enjoy content, the users simply load up their lists of shared interests to select from rather than browsing for extended periods. There is also the possibility of having accounts linked to users streaming service accounts so that when a match is made it can be added to users' watch queues within the streaming service applications themselves. Streaming service account linking is however not a concern within the first iteration of this project.

Tools and Technologies

The development of this project will require a code editor such as Atom to edit the HTML, CSS and Java Script for any web interface or alternatively platforms such as phonegap or ionic framework could be employed. A compiler (Eclipse) and relevant IDEs and plugins are required to develop Swift and Kotlin applications or applications such as asswing2app that required no coding experience could be used. Prototyping can be completed using Figma, Sketch or Adobe XD and team collaboration with Microsoft Teams, Word, Excel and Miro. A database will also need to be established to store the required data such as movie artwork, titles, list of actors, release date, runtime etc. This database would need to be cloud based and could be established using MariaDB.

    The Open Movie Database

    When developing the FlickDeck application, an open-source API like The Open Movie Database (OMDb) could be used in the searching and filtering of movie and TV shows (OMDb API, n.d.).

    Available as a web service online, the OMDb API obtains information about movies and TV shows from a database that is contributed to and maintained by its users. Obtaining a free API key will allow an application or website to search for a movie or TV show by title and return data containing the content’s year of release, content rating, runtime, genre, director and writer, cast, plot, awards & nominations, an image of the poster, and even user ratings through popular content website IMDB.

    When a user likes or swipes right in FlickDeck on a displayed movie or TV show, the OMDb API could be used to query the database for that approved content and determine key factors including genre, cast, directors & writers, and year of release. This data could then be used to develop a viewing profile and allow FlickDeck to suggest more appropriate, relatable content.

    MariaDB

    Some form of database management system (DBMS), such as the open source MariaDB server (MariaDB Foundation, n.d.), would be required to store, sort, categorise, and establish links between the compiled movie and TV show data. Available through GitHub (GitHub, Inc. 2020), MariaDB Server is a relational database tool that organises data into tables which can be linked, or related, based on common attributes. With multiple fields for each record in the database, MariaDB Server allows for quick comparison between records and to create a new table using a single query.

    As an example, a FlickDeck user may like or swipe right on a romantic-comedy movie released in 2008, starring Jane and John Doe. Based on this basic data, the MariaDB server could be queried to locate and return records that contain at least 2 of the following attributes:

    1. Year of release: 2008
    2. Genre: romance, comedy
    3. Cast: Jane Doe, John Doe

    These records would be returned as a singular table that could be used to display to the user relatable content. By storing more details about the movie or TV show and narrowing the scope of the database query, the relational database will return records that closer match the user’s viewing preferences and could help suggest more relatable content. A relational database could also compare and find similarities between multiple user’s data tables. MariaDB Server also allows for cloud integration.

    Unoffical API’s and Data Scraping

    In 2014 Netflix shutdown its public API, this creates a problem early on in the development of FlickDeck as the application has no access to an established database of the platforms current offerings to pull from in order to display user’s with intended content from the streaming platform.

    There are several unofficial API’s such as Guidebox API that offer all the data and resources that can be pulled from streaming services but for a price. These unofficial API’s are able to obtain their data through data scrapping. Data scraping is a process in which data is extracted from readable text from external sources such as information stored in the HTML code of a website ((McClendon, 2015). Internally Netflix utilises an API to transfer information to its mobile application (Netflix Data Scraping API | Stevesie Data, 2020), it is this connection that can be intercepted to obtain the API data necessary for this project.

    For a data scraping program to be able to intercept these connections the endpoints that handle the data wanted must be located. The process for this is fairly involved and requires virtualisation of the application on a computer based virtual phone operating system such as android and a network activity monitoring program such as Charles (McClendon, 2015).Using the app to go through actions that may net the access points of information we need such as clicking more information on a movie may give access to an endpoint that handles such data (McClendon, 2015). Once the appropriate points are found is when a data scraper program can be developed. Using the http URL’s that have been observed a scraper program can make pull requests from the same points to then store the data in a separate repository for the FlickDev team to access (McClendon, 2015). The implementation of such a strategy is subject to experience and resource availability as it may prove more economical for the development team to simply pay a fee to access an established API.
